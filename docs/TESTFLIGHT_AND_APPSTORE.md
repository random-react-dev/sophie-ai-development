## App Store Rejection Fix #1 (v1.0, March 10, 2026)

Four issues were raised by Apple review and fixed:

1. **Sign in with Apple name pre-fill (Guideline 4)** ‚Äî `ProfileStep.tsx` now pre-fills the name field from `user_metadata.full_name` provided by Apple Sign In.
2. **Third-party AI disclosure (Guideline 2.1 + 5.1.2(i))** ‚Äî New `AIConsentModal` component (`components/common/AIConsentModal.tsx`) shows a one-time consent prompt naming Google Gemini before Talk or Translate features activate. Consent saved to AsyncStorage.
3. **Session persistence bug (Guideline 2.1(a))** ‚Äî `startAutoRefresh()` called on startup in `client.ts`; proactive token refresh in `authStore.ts` `initialize()`; spurious SIGNED_OUT guard added to `onAuthStateChange`. Trial countdown modal disabled (no paywall exists yet).
4. **Demo account (Guideline 2.1(a))** ‚Äî Manual: create `appreview@speakwithsophie.ai` in Supabase, complete onboarding, backdate `created_at`, provide credentials in App Store Connect review info.

**App Store Connect reply for AI questions:**
1. Yes, the app uses third-party AI for analysis of data.
2. Provider: Google Gemini (via Google AI Studio API).
3. Data transmitted: Voice audio (PCM format), text transcripts, conversation history, and language learning context. No PII beyond what the user speaks in conversation.

## How to Upload a New Build to TestFlight

Every time you make changes and want to test on your iPhone via TestFlight:

### 1. Increment `buildNumber` in `app.config.ts`

Each TestFlight upload MUST have a unique build number. Increment it:
- "1" ‚Üí "2" ‚Üí "3" ‚Üí "4" ...

The `version` (e.g., "1.0.0") only needs to change for new App Store releases, not for TestFlight builds.

### 2. Regenerate iOS folder

```bash
npx expo prebuild --platform ios --clean
```

### 3. Open in Xcode

```bash
xed ios
```

### 4. Verify signing

- Select the **sophie** target
- Go to **Signing & Capabilities** tab
- Ensure **"Automatically manage signing"** is checked
- Team should be **Apexture Private Limited**
- No red errors should appear

### 5. Archive

- Set device target to **"Any iOS Device (arm64)"** or your connected iPhone (top toolbar)
- Menu: **Product > Archive**
- Wait 5-15 minutes for build to complete

### 6. Upload

- The **Organizer** window opens automatically after archive
- Click **"Distribute App"**
- Select **"App Store Connect"** ‚Üí Click **"Distribute"**
- Wait for upload to complete
- Ignore "Upload Symbols Failed" warnings ‚Äî they are harmless

### 7. Wait for processing

- Go to https://appstoreconnect.apple.com ‚Üí Your App ‚Üí **TestFlight** tab
- Build takes 5-15 minutes to process
- Once status shows **"Ready to Submit"** / **"Complete"**, the new build auto-appears in TestFlight on your iPhone

## Versioning Strategy

| Field | When to Change | Example |
|-------|---------------|---------|
| `version` | New App Store release | "1.0.0" ‚Üí "1.1.0" ‚Üí "2.0.0" |
| `buildNumber` | Every TestFlight upload | "1" ‚Üí "2" ‚Üí "3" ‚Üí "4" ... |

- Multiple builds can share the same `version` (e.g., version 1.0.0 with builds 1, 2, 3...)
- When submitting to the App Store, pick the best build from TestFlight
- `buildNumber` must ALWAYS increase (never reuse a number)

## Important Notes / Gotchas

1. **Bundle ID is permanent** ‚Äî `ai.speakwithsophie.app` cannot be changed after the first upload. A new bundle ID = a new app.
2. **"Sign in with Apple"** ‚Äî If the app has login/signup, Apple REQUIRES "Sign in with Apple" as an option. Common rejection reason.
3. **Third-party AI disclosure** ‚Äî Apple requires explicit user consent before sending data to third-party AI providers (Google Gemini). The `AIConsentModal` component handles this.
4. **Privacy Policy** ‚Äî Required for App Store submission. Host somewhere publicly accessible.
4. **Screenshots** ‚Äî Required for App Store. Take on simulator for each required device size.
5. **Never manually edit `ios/` folder** ‚Äî Always use `npx expo prebuild --platform ios --clean` to regenerate.
6. **Xcode Cloud popup** ‚Äî If it appears, click "Don't Ask Again". We do local builds only.
7. **Old bundle ID** ‚Äî The app was originally `com.fluentai.sophie` but that was taken on Apple's system. Changed to `ai.speakwithsophie.app`.
8. **Android bundle** ‚Äî Still uses `com.fluentai.sophie` in app.config.ts. Update if/when publishing to Google Play.

## Troubleshooting

### "Failed Registering Bundle Identifier"
- Bundle ID is taken. Must use a unique one. Current: `ai.speakwithsophie.app`

### "Device not registered"
- Connect iPhone via USB ‚Üí Click "Register Device" in Xcode signing

### "Communication with Apple failed"
- Connect iPhone via USB cable (not wireless)

### "Upload Symbols Failed" warnings
- Harmless. Just about dSYM debug symbols. Click Done and move on.

### Build not appearing in TestFlight
- Wait 5-15 minutes for Apple to process
- Check "Build Uploads" section in TestFlight tab for status
